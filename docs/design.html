Design & Architecture

Rhapsode

An offline-first audiobook & EPUB player that keeps your library in a Dropbox app folder — and trims the dead air out of narration so books play faster without sounding edited.

Native SwiftUI for iPhone, iPad and Mac (Catalyst). Files are downloaded in full and played from local storage; nothing streams. Dropbox is reached over its plain HTTP API so the design ports cleanly to a planned Android client.

System

Architecture at a glance

Four layers, top to bottom: SwiftUI presentation, a domain/playback layer (AVPlayer, Readium, Cadence), the library/sync/source layer behind a single LibrarySource protocol, and SwiftData persistence with on-device storage. Dropbox, Readium and CadenceKit are the only external dependencies.

Solid teal: the Dropbox boundary and the Cadence render path. Dashed: package/framework links. Everything above the source layer is backend-agnostic.

Data flow

Three pipelines

1

Library sync & download

SyncManager watches the two app-folder roots (/Audiobooks, /Books) with Dropbox longpoll, plus a manual “Scan now”. A new file is handed to BackgroundDownloader (background URLSession) which survives suspension and cold relaunch; on completion the importer parses metadata and inserts a SwiftData row. Media lands in Application Support; only relative paths are stored.

2

Cadence render-on-download

When a book finishes downloading (and Cadence resolves to on for it), CadenceRenderCoordinator analyses a mono downmix to locate silences, renders a trimmed .m4a from the original channels — chapter-chunked so memory stays bounded — and writes it to an evictable Caches file plus a TrimmedRendition row with a source↔trimmed timeline map.

3

Cross-device progress sync

Playback/reading positions are written as small JSON files under /.rhapsode-sync in the app folder (one per item, last-writer-wins by timestamp) and merged on foreground. Positions are keyed by the stable, cross-device container path — start on iPhone, resume on iPad.

Deep dive

How Cadence works

Cadence shortens silences in narration so a book plays faster without sounding edited — not hard silence-skipping, but proportional compression with an adaptive noise floor, a residual-gap floor, edge guards and equal-power crossfades at every splice.

It is a pre-render design (“Path B”): the downloaded file is analysed and a trimmed .m4a is rendered ahead of time, then played by the existing AVPlayer — no AVAudioEngine, no rewrite of the now-playing / CarPlay / AirPlay surface. The trimmed file is just an audio file, so everything that works for the original works for it.

Correctness

Source of truth

  • Original download is never modified
  • Trimmed file is a regenerable, evictable cache
  • Keyed by fingerprint + analyzer/renderer version + tier
Mapping

Source-domain positions

  • All saved positions live in source time
  • Mapped to trimmed time only at the AVPlayer boundary
  • Stable across tier changes, toggles & eviction
Analysis

Mono detect, channel render

  • Silence located on a mono downmix
  • Renderer cuts the original channels at those times
  • Zero-crossing-aligned, equal-power crossfades
